# MIXMLAAL 项目架构文档

## 1. 项目整体架构

MIXMLAAL 是一个大型全栈生态平台项目，采用模块化、多应用架构设计，支持多端开发和部署。项目架构遵循微服务理念，将不同功能模块分离为独立的应用，同时通过共享代码和服务实现模块间的协同。

### 1.1 架构层级

```

├───────────────────────┬───────────────────────┬─────────────────────────┤
```

## 2. 目录结构分析

### 2.1 核心目录结构

```
.
├── apps/                # 应用目录（核心业务模块）
├── shared/              # 共享代码（跨应用复用）
├── services/            # 微服务（独立部署的服务）
├── src/                 # 主源码（公共功能）
├── docs/                # 文档（项目文档）
├── 文档/                # 中文文档（项目文档）
├── deploy/              # 部署产物（构建后的文件）
├── scripts/             # 脚本（部署、初始化等）
├── nginx/               # Nginx配置（反向代理）
├── kubernetes/          # Kubernetes配置（容器编排）
├── observability/       # 可观测性（监控、日志）
├── miniapp/             # 小程序（多平台小程序）
├── i18n/                # 国际化（多语言支持）
├── public/              # 静态资源（公共文件）
├── assets/              # 资源文件（样式、脚本）
├── data/                # 数据文件（查询、配置）
├── test/                # 测试文件（单元测试）
├── tests/               # 测试文件（集成测试）
└── microservices/       # 微服务架构（架构文档）
```

### 2.2 应用目录结构

**apps/ 目录**：包含多个独立应用，每个应用都是一个完整的前端或后端服务

| 应用名称 | 技术栈 | 功能描述 |
|---------|-------|---------|
| admin | Vue 3 + Vite | 管理后台，用于系统管理、用户管理、权限管理等 |
| api | Node.js + Express | API服务，提供后端接口，处理业务逻辑 |
| driver | Vue 3 + Vite | 司机端应用，用于司机接单、配送管理等 |
| frontend | Vue 3 + Vite | 前端应用，面向普通用户的主应用 |
| h5 | Vue 3 + Vite | H5移动端应用，适配移动端浏览器 |
| merchant | Vue 3 + Vite | 商家端应用，用于商家管理商品、订单等 |
| react-admin | React + UmiJS | React版本的管理后台 |

### 2.3 核心服务结构

**services/ 目录**：包含独立部署的微服务

| 服务名称 | 技术栈 | 功能描述 |
|---------|-------|---------|
| auth-service | Node.js | 认证服务，处理用户认证和授权 |
| chatbot-service | Node.js | 聊天机器人服务，提供智能客服功能 |

### 2.4 共享代码结构

**shared/ 目录**：包含跨应用共享的代码和组件

| 模块名称 | 功能描述 |
|---------|---------|
| api | API客户端，封装API调用 |
| components | 共享组件，如DataTable、Modal等 |
| i18n | 国际化配置，多语言支持 |
| services | 共享服务，如分析服务、钱包服务等 |
| store | 共享状态管理，如用户状态、全局状态等 |
| utils | 工具函数，如版本管理、通用工具等 |

## 3. 核心模块说明

### 3.1 前端模块

#### 3.1.1 管理后台 (admin)
- **功能**：系统管理、用户管理、角色管理、权限管理、订单管理、商品管理等
- **技术栈**：Vue 3、Vite、Vue Router、Pinia、Tailwind CSS
- **核心页面**：仪表盘、用户管理、角色管理、权限管理、订单管理、商品管理等

#### 3.1.2 前端应用 (frontend)
- **功能**：首页、商品浏览、购物车、订单管理、个人中心、社交功能等
- **技术栈**：Vue 3、Vite、Vue Router、Pinia、Tailwind CSS
- **核心页面**：首页、商品列表、商品详情、购物车、订单、个人中心、社交等

#### 3.1.3 司机端应用 (driver)
- **功能**：订单接收、配送管理、路线导航、收入统计、个人中心等
- **技术栈**：Vue 3、Vite、Vue Router、Pinia
- **核心页面**：订单列表、订单详情、财务统计、个人信息等

#### 3.1.4 H5移动端应用 (h5)
- **功能**：适配移动端的购物、出行、配送等功能
- **技术栈**：Vue 3、Vite、Vue Router、Pinia
- **核心页面**：首页、商城、出行、配送、个人中心等

#### 3.1.5 商家端应用 (merchant)
- **功能**：商品管理、订单管理、店铺管理、营销管理等
- **技术栈**：Vue 3、Vite、Vue Router、Pinia
- **核心页面**：商品管理、订单管理、店铺信息等

#### 3.1.6 React管理后台 (react-admin)
- **功能**：基于React的管理后台，功能类似Vue版管理后台
- **技术栈**：React、UmiJS、Ant Design
- **核心页面**：仪表盘、用户管理、角色管理、权限管理等

### 3.2 后端模块

#### 3.2.1 API服务 (api)
- **功能**：提供后端接口，处理业务逻辑，连接数据库
- **技术栈**：Node.js、Express、Sequelize、Mongoose、Redis
- **核心模块**：
  - 认证模块：JWT认证、权限控制
  - 业务模块：用户、订单、商品、支付等
  - 数据模块：MySQL、MongoDB操作
  - 工具模块：缓存、加密、日志等

#### 3.2.2 开放API服务 (src/open/api)
- **功能**：提供开放API，支持第三方集成
- **技术栈**：Node.js、Express
- **核心模块**：认证、订单、支付、地图等

### 3.3 小程序模块

#### 3.3.1 多平台小程序 (miniapp)
- **功能**：支持微信、支付宝、百度等多平台小程序
- **技术栈**：原生小程序开发、多平台适配
- **核心页面**：首页、购物车、支付、出行、帮助等

## 6. 数据流分析

### 6.1 前端到后端数据流

1. **用户请求**：用户通过前端应用发起请求（如登录、购物、下单等）
2. **API调用**：前端通过Axios调用后端API
3. **认证验证**：API服务通过JWT验证用户身份和权限
4. **业务处理**：API服务处理业务逻辑
5. **数据操作**：API服务与数据库交互（MySQL、MongoDB）
6. **缓存操作**：API服务与Redis交互，处理缓存
7. **响应返回**：API服务返回处理结果给前端
8. **前端处理**：前端应用处理响应数据，更新UI

### 6.2 后端内部数据流

1. **请求接收**：Express接收HTTP请求
2. **中间件处理**：通过中间件处理认证、日志、错误等
3. **路由分发**：根据请求路径分发到对应控制器
4. **控制器处理**：控制器处理业务逻辑
5. **服务调用**：调用服务层处理复杂业务
6. **模型操作**：通过模型操作数据库
7. **工具使用**：使用工具函数处理加密、缓存等
8. **响应生成**：生成HTTP响应

## 8. 安全架构

### 8.1 前端安全
- **XSS防护**：使用Vue/React的自动转义，Content-Security-Policy
- **CSRF防护**：使用CSRF token
- **输入验证**：前端表单验证
- **敏感信息**：不在前端存储敏感信息
- **HTTPS**：使用HTTPS加密传输

### 8.2 后端安全
- **SQL注入防护**：使用Sequelize参数化查询
- **认证授权**：JWT认证、RBAC权限控制
- **密码加密**：使用bcrypt加密密码
- **API限流**：防止暴力攻击
- **日志记录**：记录关键操作和安全事件
- **依赖安全**：定期更新依赖，防止漏洞

### 8.3 基础设施安全
- **网络安全**：防火墙、网络隔离
- **服务器安全**：定期更新系统、补丁管理
- **容器安全**：Docker镜像扫描、容器隔离
- **数据备份**：定期数据备份和恢复
- **灾难恢复**：制定灾难恢复计划
