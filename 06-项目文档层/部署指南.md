## 🚀 项目简介

本项目是一个**亿级商用全平台超级综合体**，包含：
- 6大前端：Vue3 PC官网、React管理后台、HTML静态官网、H5移动端、UniApp小程序、Flutter APP、Electron桌面端
- 3大后端：Java SpringCloud微服务、Python FastAPI数据服务、Node NestJS网关+IM
- 10库数据库：分库分表架构设计
- 完整部署：Docker Compose一键部署 + Nginx商用配置

## 🖥️ 前端部署

### 1. Vue3 PC前台构建部署
```bash
cd 01-前端集群/web-vue-pc
npm install
npm run build

# 构建产物在 dist/ 目录
# 将 dist/ 目录上传到服务器或使用 Nginx 部署
```

### 2. React管理后台构建部署
```bash
cd 01-前端集群/web-react-admin
npm install
npm run build

# 构建产物在 dist/ 目录
```

### 3. Nginx静态资源配置
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # Vue3 PC前端
    location / {
        root /path/to/web-vue-pc/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    
    # React管理后台
    location /admin/ {
        alias /path/to/web-react-admin/dist;
        index index.html;
        try_files $uri $uri/ /admin/index.html;
    }
    
    # 静态文件缓存
    location ~* \.(js|css|jpg|jpeg|png|gif|ico|svg|webp)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
}
```

## 🔧 后端部署

### 1. Python FastAPI服务部署
```bash
cd 02-后端微服务集群/python-fastapi

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate     # Windows

# 安装依赖
pip install -r requirements.txt

# 启动服务
python main.py

# 生产环境使用 Gunicorn + Uvicorn
pip install gunicorn uvicorn
gunicorn -w 4 -k uvicorn.workers.UvicornWorker main:app -b 0.0.0.0:8001
```

### 2. Node NestJS服务部署
```bash
cd 02-后端微服务集群/node-nest-gateway
npm install
npm run build

# 启动服务
npm run start:prod

# 或使用 PM2 进程管理
npm install -g pm2
pm2 start dist/main.js --name nest-gateway
```

### 3. Java SpringCloud服务部署
```bash
cd 02-后端微服务集群/java-springcloud

# Maven打包
mvn clean package -DskipTests

# 运行jar包
java -jar cloud-gateway/target/cloud-gateway-1.0.0.jar
java -jar cloud-system/target/cloud-system-1.0.0.jar
# ... 其他微服务
```

## 🐳 Docker部署

### 方式一：使用 Docker Compose 一键部署
```bash
cd 05-部署运维层/docker-compose

# 启动所有服务
docker-compose -f docker-compose-full.yml up -d

# 查看服务状态
docker-compose -f docker-compose-full.yml ps

# 查看日志
docker-compose -f docker-compose-full.yml logs -f

# 停止所有服务
docker-compose -f docker-compose-full.yml down
```

### 方式二：单个服务Docker部署

#### MySQL
```bash
docker run -d \
  --name yiji-mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -v ./mysql-data:/var/lib/mysql \
  -v ../04-数据库层/mysql-sql:/docker-entrypoint-initdb.d \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

#### Redis
```bash
docker run -d \
  --name yiji-redis \
  -p 6379:6379 \
  -v ./redis-data:/data \
  redis:7 \
  redis-server --appendonly yes
```

## ⚠️ 常见问题

### Q1: npm install 安装依赖很慢？
A: 可以使用国内镜像源：
```bash
npm config set registry https://registry.npmmirror.com
```

### Q2: Python 模块安装失败？
A: 使用国内镜像源：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q3: 端口被占用？
A: Windows查找并结束占用端口的进程：
```bash
# 查找端口占用
netstat -ano | findstr :8080

# 结束进程（替换PID为实际ID）
taskkill /F /PID <进程ID>
```

### Q4: Docker服务启动失败？
A: 检查Docker Desktop是否正常运行，查看服务日志：
```bash
docker logs <container-name>
```
