# MIXMLAAL 服务清单

## 概述

本文档记录 MIXMLAAL 项目中所有的服务组件，包括后端服务、微服务、消息队列服务等。

## 二、新增服务详细说明

### 1. Kafka Consumer Service (kafka-consumer-service)

**功能**: 消费Kafka消息队列中的各类事件

**支持Topic**:
- `order.created` - 订单创建事件
- `order.paid` - 订单支付事件
- `order.completed` - 订单完成事件
- `order.cancelled` - 订单取消事件
- `payment.succeeded` - 支付成功事件
- `payment.failed` - 支付失败事件
- `payment.refunded` - 退款事件
- `user.registered` - 用户注册事件
- `user.authenticated` - 用户认证事件
- `driver.location` - 司机位置事件
- `rider.location` - 配送员位置事件
- `user.behavior` - 用户行为事件

**端口**: 3027

**依赖**:
- Kafka
- Redis
- MongoDB

---

### 2. Kafka Producer Service (kafka-producer-service)

**功能**: 生产并发送消息到Kafka队列

**端口**: 3028

**依赖**: Kafka

---

### 3. Search Service (search-service)

**功能**: 基于Elasticsearch的全文搜索服务

**索引**:
- `products` - 商品索引
- `merchants` - 商户索引
- `articles` - 文章索引

**功能**:
- 商品全文搜索（支持中文分词）
- 商户搜索（支持地理距离查询）
- 文章搜索
- 价格范围筛选
- 分类筛选
- 排序（价格、销量、评分）
- 相关性推荐

**端口**: 3029

**依赖**: Elasticsearch

---

### 4. Payment Integration Service (payment-integration-service)

**功能**: 第三方支付集成（微信支付、支付宝）

**微信支付接口**:
- 统一下单 (NATIVE)
- 订单查询
- 退款接口
- 支付回调通知处理

**支付宝支付接口**:
- 预下单 (alipay.trade.precreate)
- 订单查询 (alipay.trade.query)
- 退款接口 (alipay.trade.refund)
- 支付回调通知处理

**端口**: 3030

**环境变量**:
```
WECHAT_APP_ID
WECHAT_MCH_ID
WECHAT_API_KEY
WECHAT_NOTIFY_URL
ALIPAY_APP_ID
ALIPAY_PRIVATE_KEY
ALIPAY_PUBLIC_KEY
ALIPAY_NOTIFY_URL
```

---

### 5. SMS Service (sms-service)

**功能**: 短信验证码服务

**支持的短信服务商**:
- 阿里云短信
- 腾讯云短信

**功能**:
- 验证码发送（6位数字）
- 验证码校验
- 发送频率限制（60秒）
- 每日发送限制（10条/天）
- 多种通知类型支持

**API接口**:
```
POST /api/sms/send     - 发送验证码
POST /api/sms/verify   - 验证验证码
POST /api/sms/notify   - 发送通知
POST /api/sms/resend   - 重发验证码
```

**端口**: 3031

**环境变量**:
```
SMS_PROVIDER=aliyun|tencent
ALIYUN_ACCESS_KEY_ID
ALIYUN_ACCESS_KEY_SECRET
ALIYUN_SMS_SIGN_NAME
ALIYUN_SMS_TEMPLATE_CODE
TENCENT_SECRET_ID
TENCENT_SECRET_KEY
TENCENT_SMS_SDK_APP_ID
TENCENT_SMS_SIGN_NAME
TENCENT_SMS_TEMPLATE_ID
```

---

### 6. Service Registry (service-registry)

**功能**: 服务注册与发现中心

**功能**:
- 服务注册/注销
- 服务发现
- 心跳保活
- 负载均衡（随机）
- 故障检测

**API接口**:
```
POST /register           - 注册服务
POST /deregister         - 注销服务
POST /heartbeat         - 发送心跳
GET  /discover/:name    - 发现服务
GET  /services          - 获取所有服务
GET  /services/healthy  - 获取健康服务
GET  /health            - 健康检查
GET  /stats             - 统计信息
```

**端口**: 3032

## 三、服务间通信

### 1. Service Client SDK

**位置**: `shared/services/service-client.js`

**功能**:
- 服务注册与注销
- 心跳保活
- 服务发现
- 熔断器模式
- 重试机制
- 自动负载均衡

**使用示例**:
```javascript
const { ServiceClient } = require('../../shared/services/service-client');

const client = new ServiceClient({
  serviceName: 'my-service',
  serviceUrl: 'http://localhost:3000',
  serviceRegistryUrl: 'http://localhost:3032'
});

// 注册服务
await client.register();

// 发现并调用其他服务
const data = await client.get('user-service', '/api/users/123');

// 发送心跳
client.startHeartbeat(30000);
```

## 七、环境变量配置

所有新服务支持以下环境变量：

```bash
# 服务通用
NODE_ENV=production
PORT=<服务端口>

# Redis
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=

# MongoDB
MONGO_URI=mongodb://mongo:27017/mixmlaal

# Kafka
KAFKA_BROKER=kafka:9092

# Elasticsearch
ELASTICSEARCH_URL=http://elasticsearch:9200
ES_USERNAME=elastic
ES_PASSWORD=changeme

# 服务注册中心
SERVICE_REGISTRY_URL=http://service-registry:3032
```

## 八、监控指标

所有服务通过 `/metrics` 端点暴露Prometheus指标：

| 指标名 | 类型 | 说明 |
|--------|------|------|
| mixmlaal_http_requests_total | Counter | HTTP请求总数 |
| mixmlaal_http_request_duration_seconds | Histogram | HTTP请求延迟 |
| mixmlaal_errors_total | Counter | 错误总数 |
| mixmlaal_active_connections | Gauge | 活跃连接数 |
| mixmlaal_business_metrics | Gauge | 业务指标 |
| mixmlaal_cache_operations_total | Counter | 缓存操作数 |
| mixmlaal_queue_depth | Gauge | 队列深度 |
