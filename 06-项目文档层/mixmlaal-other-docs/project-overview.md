# MIXMLAAL 项目概览

## 1. 项目简介

MIXMLAAL 是一个完整的全栈项目，包含多个应用和服务，支持多端开发和部署。项目旨在提供一个统一的平台，整合各种功能模块，满足不同用户角色的需求。

### 1.1 项目定位

- **多端支持**：支持Web、移动端、H5等多种终端
- **模块化设计**：采用模块化架构，便于扩展和维护
- **微服务架构**：支持服务的独立部署和扩展
- **企业级应用**：满足企业级应用的性能和安全要求

### 1.2 核心价值

- **统一平台**：整合多种功能，提供一站式解决方案
- **快速开发**：提供标准化的开发流程和工具
- **可扩展性**：支持业务的快速增长和功能扩展
- **高性能**：优化系统性能，提供良好的用户体验

## 2. 项目结构

### 2.1 目录结构

```
.
├── apps/                # 应用目录
│   ├── admin/          # Vue管理后台
│   ├── api/            # Node.js API服务
│   ├── driver/         # 司机端应用
│   ├── frontend/       # 前端应用
│   ├── h5/             # H5移动端应用
│   ├── merchant/       # 商家端应用
│   └── react-admin/    # React管理后台
├── docs/               # 文档目录
├── shared/             # 共享代码
├── tests/              # 测试文件
├── .env.example        # 环境变量示例
├── package.json        # 项目配置
└── README.md           # 项目说明
```

### 2.2 应用说明

| 应用名称 | 技术栈 | 功能描述 |
|---------|-------|---------|
| admin | Vue 3 + Vite | 管理后台，用于管理系统配置、用户、角色、权限等 |
| api | Node.js + Express | API服务，提供后端接口，处理业务逻辑 |
| driver | Vue 3 + Vite | 司机端应用，用于司机接单、配送管理等 |
| frontend | Vue 3 + Vite | 前端应用，面向普通用户的主应用 |
| h5 | Vue 3 + Vite | H5移动端应用，适配移动端浏览器 |
| merchant | Vue 3 + Vite | 商家端应用，用于商家管理商品、订单等 |
| react-admin | React + UmiJS | React版本的管理后台 |

## 3. 技术栈

### 3.1 后端技术栈

- **Node.js**：JavaScript运行时环境
- **Express.js**：Web应用框架
- **MySQL**：关系型数据库
- **MongoDB**：文档型数据库
- **Redis**：缓存服务
- **Sequelize**：ORM框架
- **Mongoose**：MongoDB对象模型工具
- **JWT**：认证和授权

### 3.2 前端技术栈

- **Vue 3**：前端框架
- **React**：前端框架
- **Vite**：构建工具
- **UmiJS**：React应用框架
- **Tailwind CSS**：CSS框架
- **Ant Design**：UI组件库
- **Pinia**：状态管理
- **Vue Router**：路由管理
- **Axios**：HTTP客户端

### 3.3 部署技术栈

- **Docker**：容器化部署
- **Kubernetes**：容器编排
- **Nginx**：Web服务器和反向代理
- **PM2**：进程管理
- **Let's Encrypt**：SSL证书

## 5. 架构设计

### 5.1 系统架构

MIXMLAAL 采用分层架构设计，包括：

- **表示层**：前端应用，负责用户界面
- **业务逻辑层**：API服务，处理业务逻辑
- **数据访问层**：数据库操作，数据持久化
- **基础设施层**：服务器、网络、存储等

### 5.2 数据流

1. **前端请求**：用户通过前端应用发起请求
2. **API处理**：API服务接收请求，处理业务逻辑
3. **数据操作**：API服务与数据库交互，执行数据操作
4. **响应返回**：API服务返回处理结果给前端
5. **前端展示**：前端应用展示数据给用户

### 5.3 安全架构

- **认证**：基于JWT的认证系统
- **授权**：基于角色的权限控制
- **加密**：敏感数据加密存储
- **防护**：防止SQL注入、XSS等攻击
- **审计**：记录系统操作日志

## 6. 开发流程

### 6.1 开发环境搭建

1. **克隆代码**：`git clone <项目地址>`
2. **安装依赖**：`npm install && npm run install:all`
3. **配置环境变量**：复制 `.env.example` 为 `.env` 并配置
4. **启动开发服务器**：`npm run dev:api` 等

### 6.2 代码规范

- **ESLint**：代码风格检查
- **Prettier**：代码格式化
- **Git**：版本控制
- **Commit规范**：遵循语义化提交

### 6.3 测试流程

- **单元测试**：测试单个功能模块
- **集成测试**：测试模块间的交互
- **端到端测试**：测试完整的业务流程

### 6.4 部署流程
