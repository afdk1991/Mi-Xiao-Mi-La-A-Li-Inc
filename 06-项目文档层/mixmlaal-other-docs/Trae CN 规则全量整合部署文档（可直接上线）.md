# 五、高级语法全套直接用法（提升管控力）

- **\-\-\-always**：强制常驻生效，适合核心红线规范，全程不掉效

- **\-\-\-when**：按文件、按目录精准触发，多文件差异化管控专用

- **\!override**：强制覆盖个人规则，团队统一标准必备，杜绝个性化偏差

- **\!include**：拆分大型规则，模块化管理，方便长期迭代维护

- **\#Rule**：对话手动唤醒规则，临时整改、紧急规范一键启用

# 六、全维度落地硬性细节（不漏坑、不失效）

## 6\.1 文件层面硬性要求（必守）

- 文件名固定不变，大小写严格匹配，禁止私自改名、简写、乱改后缀。

- 文件编码强制UTF\-8，拒绝GBK，防止乱码识别失败、规则静默失效。

- 全部路径统一使用项目根目录相对路径，绝对路径一律禁止使用。

- \.trae规则目录绝对不允许加入gitignore，必须全程纳入版本管控。

- 文件内只写规则正文，不贴截图、不堆冗余备注、不搞特殊符号。

## 6\.2 规则编写落地细节

- 不用模糊口语，不用尽量、大概、建议，全部改用必须、禁止、统一量化标准。

- 一条规则只写一件事，不堆砌多条要求，避免AI拆分混乱、执行跑偏。

- 所有头尾标记必须闭合，少一个闭合符直接整条规则作废失效。

- 同项目规则之间无冲突，不反向要求格式，避免AI逻辑错乱。

## 6\.3 生效核验落地细节

- 改完规则必开新对话，旧对话缓存不清，规则永远不生效。

- 看左上角规则标识，有标识代表加载成功，无标识立刻排错。

- 终端执行校验命令，一键自查语法、路径、冲突三类问题。

## 6\.4 团队推行落地细节

- 权限分级管控：仅负责人可改规则，开发只准查看使用。

- 规则更新全员同步，统一拉代码、统一重启、统一生效时间。

- 接入CI/CD自动校验，不合规代码直接拦截，不用人工盯守。

# 七、故障极速排查全套方案（规则不生效直接照做）

1. 第一步：重启Trae CN，清空软件全局缓存，刷新运行环境。

2. 第二步：核对文件名、目录、编码三件套，排查基础低级错误。

3. 第三步：新建空白对话，极简规则单条测试，隔离问题范围。

4. 第四步：打开日志面板搜索rules，查看后台真实报错原因。

5. 第五步：逐段删减规则，定位冲突、冗余、语法错误条目。
