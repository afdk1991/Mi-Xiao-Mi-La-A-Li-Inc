# MIXMLAAL 项目部署方案

## 1. 环境准备

### 1.1 服务器要求
- **操作系统**: Linux (Ubuntu 20.04 LTS 或 CentOS 7+)
- **CPU**: 至少 2 核
- **内存**: 至少 4GB
- **存储**: 至少 50GB
- **网络**: 具备公网IP，开放必要端口（3000、8080等）

### 1.2 软件依赖
- **Node.js**: v16.0.0 或更高版本
- **MySQL**: 8.0 或更高版本
- **MongoDB**: 4.4 或更高版本
- **Redis**: 6.0 或更高版本
- **Nginx**: 1.18 或更高版本（用于反向代理）

## 2. 数据库配置

### 2.1 MySQL 配置
1. 安装 MySQL
   ```bash
   # Ubuntu
   sudo apt update
   sudo apt install mysql-server
   
   # CentOS
   sudo yum install mysql-server
   ```

2. 创建数据库
   ```sql
   CREATE DATABASE mixmlaal CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   CREATE USER 'mixmlaal'@'localhost' IDENTIFIED BY 'your_password';
   GRANT ALL PRIVILEGES ON mixmlaal.* TO 'mixmlaal'@'localhost';
   FLUSH PRIVILEGES;
   ```

### 2.2 MongoDB 配置
1. 安装 MongoDB
   ```bash
   # Ubuntu
   sudo apt update
   sudo apt install mongodb
   
   # CentOS
   sudo yum install mongodb
   ```

2. 启动 MongoDB 服务
   ```bash
   sudo systemctl start mongodb
   sudo systemctl enable mongodb
   ```

### 2.3 Redis 配置
1. 安装 Redis
   ```bash
   # Ubuntu
   sudo apt update
   sudo apt install redis-server
   
   # CentOS
   sudo yum install redis
   ```

2. 启动 Redis 服务
   ```bash
   sudo systemctl start redis
   sudo systemctl enable redis
   ```

## 3. API 服务部署

### 3.1 部署步骤
1. 克隆代码仓库
   ```bash
   git clone <repository_url> /opt/mixmlaal
   cd /opt/mixmlaal
   ```

2. 安装依赖
   ```bash
   cd apps/api
   npm install
   ```

3. 配置环境变量
   ```bash
   cp .env.example .env
   # 编辑 .env 文件，配置数据库连接等信息
   ```

4. 启动 API 服务
   ```bash
   # 开发环境
   npm run dev
   
   # 生产环境
   npm start
   ```

### 3.2 进程管理
使用 PM2 管理 API 服务进程

1. 安装 PM2
   ```bash
   npm install -g pm2
   ```

2. 启动服务
   ```bash
   pm2 start src/app.js --name "mixmlaal-api"
   ```

3. 设置开机自启
   ```bash
   pm2 startup
   pm2 save
   ```

## 4. 前端项目部署

### 4.1 构建步骤
1. 安装依赖
   ```bash
   cd /opt/mixmlaal/apps/admin
   npm install
   ```

2. 构建项目
   ```bash
   npm run build
   ```

3. 部署构建产物
   将 `dist` 目录下的文件复制到 Nginx 网站根目录
   ```bash
   sudo cp -r dist/* /var/www/html/
   ```

### 4.2 Nginx 配置
1. 编辑 Nginx 配置文件
   ```bash
   sudo nano /etc/nginx/sites-available/mixmlaal
   ```

2. 配置反向代理
   ```nginx
   server {
       listen 80;
       server_name your_domain.com;

       # 前端静态文件
       location / {
           root /var/www/html;
           index index.html;
           try_files $uri $uri/ /index.html;
       }

       # API 反向代理
       location /api/ {
           proxy_pass http://localhost:3000/;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

3. 启用配置
   ```bash
   sudo ln -s /etc/nginx/sites-available/mixmlaal /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   ```

## 5. 容器化部署（Docker）

### 5.1 Docker Compose 配置
创建 `docker-compose.yml` 文件

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: mixmlaal-mysql
    environment:
      MYSQL_ROOT_PASSWORD: your_root_password
      MYSQL_DATABASE: mixmlaal
      MYSQL_USER: mixmlaal
      MYSQL_PASSWORD: your_password
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    restart: always

  mongodb:
    image: mongo:4.4
    container_name: mixmlaal-mongodb
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db
    restart: always

  redis:
    image: redis:6.0
    container_name: mixmlaal-redis
    ports:
      - "6379:6379"
    restart: always

  api:
    build:
      context: ./apps/api
      dockerfile: Dockerfile
    container_name: mixmlaal-api
    ports:
      - "3000:3000"
    depends_on:
      - mysql
      - mongodb
      - redis
    restart: always

  admin:
    build:
      context: ./apps/admin
      dockerfile: Dockerfile
    container_name: mixmlaal-admin
    ports:
      - "80:80"
    depends_on:
      - api
    restart: always

volumes:
  mysql_data:
  mongodb_data:
```

### 5.2 Dockerfile 配置

#### API 服务 Dockerfile
```dockerfile
FROM node:16-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm", "start"]
```

#### 前端项目 Dockerfile
```dockerfile
FROM node:16-alpine as build

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

FROM nginx:1.18-alpine

COPY --from=build /app/dist /usr/share/nginx/html

COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 5.3 启动容器
```bash
docker-compose up -d
```

## 6. 监控和日志

### 6.1 日志管理
- **API 服务日志**: 使用 PM2 日志或 Docker 日志
  ```bash
  # PM2 日志
  pm2 logs mixmlaal-api
  
  # Docker 日志
  docker logs mixmlaal-api
  ```

- **Nginx 日志**: 位于 `/var/log/nginx/` 目录

### 6.2 监控工具
- **Prometheus**: 监控系统指标
- **Grafana**: 可视化监控数据
- **ELK Stack**: 日志收集和分析

## 8. 安全配置

### 8.1 服务器安全
- 配置防火墙，只开放必要端口
- 定期更新系统和软件
- 使用 SSH 密钥认证，禁用密码登录
- 安装 Fail2ban 防止暴力破解

### 8.2 应用安全
- 使用 HTTPS 加密传输
- 配置 CORS 策略
- 实现速率限制防止 API 滥用
- 定期更新依赖包，修复安全漏洞

## 9. 故障排查

### 9.1 常见问题
1. **API 服务启动失败**
   - 检查数据库连接配置
   - 检查 Redis 连接状态
   - 查看 PM2 日志

2. **前端页面无法访问**
   - 检查 Nginx 配置
   - 检查前端构建产物是否正确部署
   - 查看 Nginx 日志

3. **数据库连接失败**
   - 检查数据库服务是否运行
   - 检查数据库连接配置
   - 检查数据库用户权限

### 9.2 监控告警
设置监控告警，及时发现和处理问题
- 服务可用性监控
- 数据库性能监控
- 系统资源监控
